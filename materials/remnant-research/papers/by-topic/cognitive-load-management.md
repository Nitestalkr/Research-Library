#### [Context, Reasoning, and Hierarchy: Cost-Performance Study of Compound LLM Agent Design](https://arxiv.org/abs/2605.16205)

- **arXiv:** 2605.16205
- **Published:** 2026-05-15
- **Relevance:** GNW Phase 6 (Coordination) + GRAO (agent optimization)

**Why it matters:** Programmatic state abstraction delivers largest returns per token (76% improvement). Distributing deliberation tools across hierarchy degrades performance (deliberation cascade — 3.4× worse). This is a cognitive load management finding: hierarchical decomposition without deliberation achieves best performance. Cognitive load is a first-class optimization variable.

**Framework relevance score:** 8/10

#### [Confirming Correct, Missing the Rest: LLM Tutoring Agents Struggle Where Feedback Matters](https://arxiv.org/abs/2605.16207)

- **Published:** 2026-05-15

## Key Research Questions for This Domain

1. **What is the cognitive load capacity of LLM-based agents?** — Context window limits are the hard boundary, but what about the soft boundary of "cognitive overload" where reasoning degrades?
2. **How do agents manage cognitive load across tasks?** — GNW's drive competition is essentially cognitive load arbitration. What papers address this?
3. **Can cognitive load be measured as a capability metric?** — Agent Psychometrics (2604.00594) decomposes agent ability — cognitive load management might be a decomposable component.
4. **What is the optimal cognitive load distribution across multi-agent fleets?** — OrgAgent shows hierarchy reduces cognitive load by 74.52%. What's the optimal distribution?
